Công cụ visuallize
1.Tổng quan
-Việc visualize trong đồ án “Khảo sát phân đoạn ảnh dựa trên câu nhắc trực quan (SAM) là bước quan trọng để: 
    o Hiển thị kết quả mask hoặc vùng phân đoạn trên ảnh gốc.
    o Kiểm tra độ chính xác của mô hình.
    o Cho phép người dùng tương tác bằng visual prompt (click chọn điểm, vẽ hộp hoặc mask thô).
2.Chi tiết 
2.1 OpenCV
-Thư viện mã nguồn mã phổ biến trong xử lý ảnh và thị giác máy tính.
-Tính năng: Đọc, ghi ảnh, hiển thị ảnh, vẽ mask overlay, nhận input từ chuột.
-Ưu điểm:
    o Dễ kết hợp với SAM.
    o Nhận input visual promt trực tiếp.
    o Hiển thị nhanh, phù hợp demo.
-Nhược điểm: Giao diện cơ bản, không web, ít tùy biến giao diện đẹp.
2.2 Matplotlib
-Visualization trong notebook hoặc báo cáo.
-Tính năng:
    o Overlay mask và alpha blending.
    o Hỗ trợ color map, chú thích, title, legend.
    o Lưu ảnh đẹp ra PNG, PDF.
-Ưu điểm:
    o Visualization đẹp, dễ tùy chình màu và style.
    o Phù hợp báo cáo.
    o Dễ overlay nhiều lớp, nhiều màu.
-Nhược điểm:
    o Không nhận input trực tiếp từ người dùng (không click point)
    o Chậm khi xử lý nhiều ảnh.
2.3 Streamlit
-Tạo web UI đơn giản cho mô hình.
-Tính năng chính:
    o Upload ảnh, hiển thị ảnh + mask overlay.
    o Slider, button để điểu chỉnh alpha hoặc màu mask.
-Ưu điểm:  
    o Giao diện web đẹp, dễ triển khai demo.
    o Thích hợp chia sẻ trực tuyến.
-Nhược điểm:
    o Không hỗ trợ vẽ trực tiếp trên ảnh (Không click point/draw box)
    o Chạy chậm hơn OpenCV nếu ảnh lớn.
2.4 Gradio
-Demo AI tương tác.
-Tính năng:
    o Upload ảnh, click vùng (point/box), preview mask trực tiếp.
    o Tương tác trực tiếp với mô hình, hiển thị mask overlay.
-Ưu điểm:
    o Tương tác trực tiếp, gần giống OpenCV nhưng web-based.
    o Demo AI nhanh.
-Nhược điểm:
    o Cần cài thêm, layout ít linh hoạt hơn Streamlit.
3.Lựa chọn
-Nếu không cần web UI, chỉ visualize mask, overlay và kiểm tra kết quả mô hình SAM, thì chọn OpenCV:
    o Dễ cài đặt.
    o Chạy trực tiếp trên desktop, không cần sever hay trình duyệt.
    o Hỗ trợ alpha blending, overlay, hiển thị nhanh kết quả.
    o Có thể kết hợp với Matplotlib để có giao diện đẹp và tạo subplot giúp so sánh nhiều mask cùng lúc.
Link code tham khảo: https://docs.opencv.org/4.x/db/d75/samples_2cpp_2create_mask_8cpp-example.html?utm_source=chatgpt.com


4. Ngôn ngữ
4.1. Python
- Vai trò: Xử lý dữ liệu, chạy mô hình segmentation, kết nối với phần visualize.
- Ưu điểm:
Có hệ sinh thái AI mạnh mẽ (PyTorch, TensorFlow, OpenCV).
Dễ dàng xuất kết quả (mask, bounding box, contour) sang định dạng JSON, PNG, hoặc NumPy để visualize.
- Các thư viện liên quan:
OpenCV, Matplotlib, Pillow: xử lý và hiển thị ảnh.
PyTorch, TorchVision, Segment Anything (SAM): mô hình phân đoạn.
Streamlit, Gradio, Dash, Plotly: xây dựng giao diện web trực quan hóa kết quả.
NumPy, Pandas: xử lý dữ liệu trung gian.

4.2. JavaScript (HTML/CSS)
- Vai trò: Hiển thị trực quan trên giao diện web, nhận dữ liệu từ backend (Python hoặc Node.js).
- Thư viện/Framework sử dụng:
Three.js: hiển thị dữ liệu phân đoạn 3D, mask không gian hoặc đám mây điểm.
D3.js: hiển thị biểu đồ hoặc phân bố vùng phân đoạn.
Canvas API / WebGL: render ảnh gốc và mask trực tiếp trên trình duyệt, cho phép người dùng tương tác (phóng to, di chuyển, thay đổi opacity).
React.js hoặc Vue.js: nếu giao diện web phức tạp, có thể xây dựng ứng dụng frontend hiện đại, dễ mở rộng.

4.3. C++
- Vai trò:
Xử lý hiệu năng cao hoặc tích hợp mô hình segmentation trong các ứng dụng chạy offline.
Dùng cùng với OpenCV C++, OpenGL, hoặc Qt để visualize ảnh và mask.
- Các thư viện có thể dùng:
OpenCV (C++ API): xử lý ảnh, overlay mask, hiển thị kết quả.
OpenGL hoặc Vulkan: dựng hình và hiển thị vùng segmentation 2D/3D tốc độ cao.
Qt: xây dựng giao diện đồ họa desktop (GUI) hỗ trợ người dùng nhập ảnh và xem kết quả.

5. Ngôn ngữ nên dùng
- Trong segmentation, python là ngôn ngữ được khuyến khích (cơ mà toi ngu python ae ạ) vì
- Hỗ trợ thư viện phong phú, mạnh mẽ cho xử lý ảnh và học sâu, bao gồm OpenCV, Pillow, NumPy, PyTorch, TensorFlow, và các thư viện chuyên biệt cho segmentation như segment-anything (SAM), Detectron2, MMsegmentation.
- Tích hợp với môi trường thử nghiệm nhanh: Python tương thích với Jupyter Notebook và Google Colab, cho phép visualize kết quả trực tiếp, thử nghiệm các mask hoặc overlay ảnh mà không cần biên dịch.
- Hỗ trợ cộng đồng và tài liệu: Hầu hết các mô hình segmentation và ví dụ demo đều có code Python, giúp sinh viên hoặc nhóm nghiên cứu dễ dàng tham khảo và triển khai.

6. Công cụ hỗ trợ

Google Colab là một môi trường lập trình dựa trên web, cho phép viết và chạy code Python trực tiếp trên trình duyệt mà không cần cài đặt bất kỳ phần mềm nào trên máy tính. Đây là công cụ rất phù hợp cho các dự án xử lý ảnh và segmentation nhờ các đặc điểm sau:
Chạy code Python miễn phí trên cloud: Colab cung cấp GPU/TPU miễn phí (hoặc nâng cấp trả phí để dùng GPU mạnh hơn), giúp việc huấn luyện mô hình segmentation hoặc xử lý ảnh lớn nhanh hơn so với máy cá nhân.
Dễ dàng thử nghiệm và visualize: Colab hỗ trợ trực tiếp hiển thị hình ảnh, mask, overlay hoặc biểu đồ bằng các thư viện như matplotlib, OpenCV hay Plotly. Điều này rất tiện khi muốn quan sát kết quả phân đoạn trung gian mà không cần biên dịch hay cài đặt phức tạp.
Tích hợp sẵn các thư viện phổ biến: Hầu hết các thư viện machine learning và computer vision như PyTorch, TensorFlow, OpenCV, segment-anything đều có thể cài đặt hoặc đã được tích hợp sẵn, giúp tiết kiệm thời gian thiết lập môi trường.
Hỗ trợ chia sẻ và hợp tác: Colab lưu file dưới dạng Notebook, dễ dàng chia sẻ, cho phép nhiều người cùng chỉnh sửa và chạy thử nghiệm song song.
